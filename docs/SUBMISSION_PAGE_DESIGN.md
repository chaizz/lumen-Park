# Lumen Park - 投稿页面 (Submission Page) 产品设计

## 1. 概述
投稿页面是用户分享摄影作品和胶片配方的入口。设计的核心目标是**简化复杂的参数录入过程**，通过自动化解析 EXIF 数据来减少用户负担，同时提供直观的配方编辑界面。

## 2. 页面流程
1.  **上传图片**：用户拖拽或点击上传一组图片（最多9张）。
2.  **列表编辑**：
    *   显示图片缩略图列表。
    *   点击某张图片，右侧显示该图片的详细参数表单。
3.  **智能解析**：系统自动读取每张图片的 EXIF 信息。
4.  **参数录入**：
    *   用户可以为每张图片单独设置配方。
    *   **“应用到所有”功能**：支持将当前图片的配方一键应用到其他图片。
5.  **补充信息**：填写整个帖子的标题、描述、标签。
6.  **发布**：提交至服务器。

## 3. 详细功能模块

### 3.1 图片上传区域 (Image Uploader)
*   **交互**：支持多选上传。
*   **列表展示**：
    *   上传后以网格形式展示缩略图。
    *   支持拖拽排序（第一张为封面）。
    *   支持删除。
    *   当前选中的图片高亮显示。

### 3.2 参数编辑区 (右侧/下方)
*   **联动**：点击左侧图片列表中的某一张，右侧区域刷新为该图片的数据。
*   **EXIF 数据**：每张图片独立。
*   **富士配方**：每张图片独立。
*   **快捷操作**：
    *   “复制当前配方到所有图片”按钮。

### 3.3 基本信息 (Basic Info)
*   属于 Post 层级，所有图片共用。
*   **标题**
*   **描述**
*   **标签**
*   **展示方式**：卡片式展示，默认自动填充，支持手动修改。
*   **字段**：
    *   相机型号 (Camera Model)
    *   镜头型号 (Lens Model)
    *   光圈 (Aperture)
    *   快门 (Shutter Speed)
    *   ISO
    *   焦距 (Focal Length)

### 3.4 富士配方 (Fujifilm Recipe)
*   **设计风格**：模仿相机菜单的 UI 风格。
*   **字段**：
    *   **胶片模拟 (Film Simulation)**：下拉选择 (Classic Chrome, Nostalgic Neg 等)。
    *   **动态范围 (Dynamic Range)**：DR100/200/400/Auto。
    *   **颗粒效果 (Grain)**：强/弱 + 大/小。
    *   **白平衡 (White Balance)**：模式 + 色温 + R/B 偏移（提供可视化坐标轴选择器）。
    *   **高光/阴影 (Highlight/Shadow)**：-2 到 +4 滑块或步进器。
    *   **色彩 (Color)**：-4 到 +4。
    *   **锐度 (Sharpness)**：-4 到 +4。
    *   **清晰度 (Clarity)**：-5 到 +5。

### 3.5 底部操作栏
*   **发布 (Publish)**：提交表单。
*   **存草稿 (Save Draft)**：保存当前状态以便稍后编辑。
*   **取消**：返回上一页。

## 4. API 需求

### 4.1 图片上传
*   `POST /api/v1/upload/image`: 上传图片，返回 URL 和解析出的 EXIF/MakerNote 数据。

### 4.2 发布帖子
*   `POST /api/v1/posts/`: 提交完整的帖子数据（包含 User, Post, Exif, Recipe 信息）。

## 5. UI/UX 规范
*   **表单验证**：必填项校验，错误提示高亮。
*   **响应式**：移动端适配单栏布局，PC 端可采用双栏布局（左图右文）。
*   **Loading 状态**：发布过程中禁用按钮并显示 Loading。
