# Lumen Park - 首页 (Home Page) 产品设计

## 1. 概述
首页是用户探索和发现内容的中心。通过**瀑布流 (Waterfall Layout)** 布局和 **滚动刷新 (Infinite Scroll)** 机制，提供沉浸式的内容浏览体验。

## 2. 页面布局
- **导航栏 (Top Nav)**：固定在顶部，包含Logo、搜索、投稿入口、用户头像等。
- **内容区域 (Content)**：全屏宽度的瀑布流容器。

## 3. 核心功能

### 3.1 瀑布流展示
- **布局方式**：采用 **JS 控制的 Masonry 布局**（Mobile: 2列, Tablet: 3列, Desktop: 4列）。
  - **特点**：确保图片从左至右、从上至下均匀填充，避免 CSS Columns 造成的列间高度差异过大或底部留白问题。
- **图片卡片**：
  - **封面图**：展示作品的封面（如果是多图，显示多图标记）。
  - **信息展示**：
    - **标题**：限制显示的行数（如2行），超出显示省略号。
    - **用户信息**：头像 + 昵称。
    - **互动数据**：仅展示 **浏览量 (Views)** 和 **点赞量 (Likes)**。
      - **不展示**：发布时间、评论数等其他信息，以保持界面简洁，聚焦视觉内容。

### 3.2 滚动刷新 (Infinite Scroll)
- **触发机制**：当用户滚动到页面底部（距离底部一定阈值，如200px）时，自动加载下一页内容。
- **加载状态**：
  - **加载中**：底部显示 Loading 动画。
  - **加载完成**：追加新内容到列表末尾。
  - **无更多内容**：显示“没有更多了”提示。
- **分页策略**：后端接口支持 `skip` 和 `limit` 参数，每次加载 20 条数据。

## 4. API 需求
- `GET /api/v1/posts/`
  - **参数**：
    - `skip`: 偏移量 (Integer)
    - `limit`: 每页数量 (Integer, Default: 20)
  - **返回**：包含 Post 对象列表，每个对象需包含 `views_count` 和 `likes_count`。

## 5. UI/UX 细节
- **图片加载**：使用懒加载 (Lazy Load) 技术，提升首屏速度。
- **过渡动画**：新加载的卡片通过淡入 (Fade In) 动画出现。
- **悬停效果**：鼠标悬停在卡片上时，图片轻微放大或阴影加深，增强点击欲望。
